# PAD问题快速诊断系统

一个用于快速定位和收集用户PAD问题的客服反馈系统。

## 功能特点

### 📋 核心功能
- **UID输入**：快速识别用户身份
- **问题分类**：区分硬件和软件问题
- **智能引导**：根据问题类型提供相应解决方案
- **状态追踪**：记录问题是否已解决
- **数据收集**：本地存储所有反馈记录

### 🔧 硬件问题支持
- 维修
- 申请备用机
- 换新
- 电量不足
- 点阵笔无法安装
- 其他（自定义描述）

**特别提示**：硬件问题会自动显示"请走班主任系统硬件售后"提醒

### 💻 软件问题支持
- 学习计划无法排
- 更新激活码
- 查看学习统计（每日21:00查看）
- 其他（自定义描述）

### ✅ 问题解决流程
1. 提交诊断后，选择问题状态
2. **已解决**：记录成功解决的案例
3. **未解决**：自动生成转人工处理的工单格式
   - 格式：`UID xxx 存在「硬件/软件」xxx问题`

## 使用方法

> 📘 **新手？** 查看 [快速开始指南](QUICK_START.md) 5分钟上手！

### 快速开始

1. **直接打开使用**
   ```bash
   # 方式1：直接在浏览器中打开
   open index.html

   # 方式2：使用本地服务器
   python3 -m http.server 8000
   # 然后访问 http://localhost:8000
   ```

2. **填写表单**
   - 输入用户UID
   - 选择问题类型（硬件/软件）
   - 选择具体问题或填写自定义描述
   - 提交诊断

3. **确认解决状态**
   - 选择"已解决"或"未解决，转人工处理"
   - 系统自动生成结果

4. **复制结果**
   - 点击"复制结果"按钮
   - 粘贴到工单系统或聊天工具

## 数据管理

### 本地存储
所有反馈记录自动保存在浏览器的 `localStorage` 中，包含：
- 用户UID
- 问题类型
- 问题详情
- 解决状态
- 时间戳

### 导出记录
在浏览器控制台中运行：
```javascript
exportRecords()
```
将下载包含所有记录的JSON文件。

### 查看记录
在浏览器控制台中运行：
```javascript
JSON.parse(localStorage.getItem('feedbackRecords'))
```

### 清除记录
```javascript
localStorage.removeItem('feedbackRecords')
```
